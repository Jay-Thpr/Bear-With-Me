# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation + Research Pipeline.
> Generated by gsd-ui-researcher. Covers UI-01 (skill selection / research phase screen).

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — raw Tailwind CSS |
| Preset | not applicable |
| Component library | none — native HTML elements styled with Tailwind utility classes |
| Icon library | none — text labels and emoji fallbacks only for Phase 1 |
| Font | Inter via `next/font/google` (system fallback: ui-sans-serif, system-ui) |

**Rationale (from RESEARCH.md + PROJECT.md):** Stack is locked to Tailwind CSS. No shadcn initialized — project is pre-scaffolding. For a 24-hour hackathon with a single Phase 1 screen, shadcn initialization adds friction with no payoff at this phase. Executor should initialize shadcn before Phase 2 (session UI has more component complexity). Phase 1 uses plain Tailwind throughout.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline icon-to-label padding |
| sm | 8px | Input internal padding (top/bottom), label-to-input gap |
| md | 16px | Button horizontal padding, input horizontal padding, gap between stacked elements |
| lg | 24px | Gap between heading and input group |
| xl | 32px | Gap between input group and status area |
| 2xl | 48px | Vertical rhythm between major sections |
| 3xl | 64px | Centering offset — min vertical padding from viewport edge |

Exceptions:
- Touch target minimum: 44px height on the primary button (WCAG 2.5.5 minimum touch target)
- Input height: 44px (aligned with touch target)

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 32px (text-3xl) | 700 (font-bold) | 1.2 | Page heading "What do you want to learn?" |
| Body | 16px (text-base) | 400 (font-normal) | 1.5 | Input placeholder, status copy, link text |
| Label | 14px (text-sm) | 700 (font-bold) | 1.4 | Button label, error message, step indicator |

**Font:** Inter loaded via `next/font/google`. Falls back to `ui-sans-serif, system-ui, sans-serif`.

**Constraint:** Phase 1 is a single full-screen centered page. Only 3 type roles are needed — Display for the heading, Body for supporting copy and status, Label for the action button. Label is visually distinct from Body via size (14px vs 16px) and context, so both Display and Label share weight 700 with no ambiguity.

## Color

This is a dark-mode-first UI (hackathon/demo context — coaching tools are typically dark). All values are Tailwind utility class descriptors, not hex codes, to align with Tailwind CSS locked stack.

| Role | Tailwind Value | Approximate Hex | Usage |
|------|---------------|-----------------|-------|
| Dominant (60%) | `bg-zinc-950` | #09090b | Full page background, main surface |
| Secondary (30%) | `bg-zinc-900` | #18181b | Input field background, status panel background |
| Accent (10%) | `bg-blue-600` | #2563eb | Primary button background only |
| Destructive | `bg-red-600` | #dc2626 | Error state text/icon — error message copy only |

**Accent reserved for:** Primary CTA button ("Start Research") background only. No other element in Phase 1 uses accent color. Status text, links, and indicators use neutral zinc tones.

**Text colors:**
- Primary text: `text-zinc-50` (#fafafa) — headings, input value text
- Secondary text: `text-zinc-400` (#a1a1aa) — placeholder text, supporting copy
- Error text: `text-red-400` (#f87171) — error state message
- Success link: `text-blue-400` (#60a5fa) — "View Skill Document" link (done state)

## Interaction States

Phase 1 has one primary interaction: skill name entry + research trigger. The state machine is:

| State | Visual |
|-------|--------|
| `idle` | Input empty or with value; button active (if input non-empty) |
| `researching` | Button disabled + text changes to step label; status panel visible below button |
| `done` | Status panel shows success message + Google Docs link |
| `error` | Status panel shows error message in `text-red-400` |

**Researching sub-states (three-step pipeline):**

The pipeline has 3 steps (from RESEARCH.md Open Question #2 — step-by-step progress makes demo more legible). Show each step as it begins:

| Step | Status Text |
|------|-------------|
| Step 1 (grounding search) | "Finding tutorials..." |
| Step 2 (video analysis) | "Analyzing videos..." |
| Step 3 (synthesis + Docs write) | "Writing skill document..." |

Each step label replaces the previous. A neutral spinner (animated border, `text-zinc-500`) precedes the label.

**Button disabled state:** `opacity-50 cursor-not-allowed` (Tailwind defaults). Disabled when: input is empty OR state is `researching`.

## Accessibility

- Input and button share a common width (`w-80` = 320px) for visual alignment.
- Input has `aria-label="Enter a skill to research"`.
- Button has `type="button"` explicitly; Enter key on input also triggers research.
- Status panel role: `aria-live="polite"` — screen reader announces step changes without interrupting.
- Color contrast: `text-zinc-50` on `bg-zinc-950` exceeds WCAG AA (contrast ratio ~19:1). `text-red-400` on `bg-zinc-900` meets AA for normal text at 14px+ (ratio ~5:1).
- Minimum touch target: button and input both 44px tall.

## Layout

Single full-screen centered page. No navigation, no sidebar, no header. The entire Phase 1 UI is one screen:

```
[full viewport — bg-zinc-950]
       [centered column — max-w content, items-center]
              "What do you want to learn?"
              [input 320px wide]
              [button 320px wide]
              [status panel 320px wide — visible when not idle]
```

No responsive breakpoints are needed for Phase 1 — this is a desktop demo (hackathon constraint, no mobile requirement).
